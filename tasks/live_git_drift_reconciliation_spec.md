## Remaining Rule From The Spec

Synthetic proposal materials are generated by proposal logic and should not be
treated as required source rows in `job_materials`.

Validation should skip `synthetic_*` identifiers when checking proposal body
materials against job source materials:

- `item_code`
- `id`
- `material_id`

This prevents generated rows such as `synthetic_jolly_tub` from blocking PDF
generation with a stale/missing source-material 409.

## Operational Guidance

- Do not deploy from a stale tree.
- Before deploying, confirm `main` includes both:
  - Sun Valley Phase 8 alignment work
  - Rules/audit and PDF validation work
- If Sun Valley is regenerated, verify:
  - deleted bundles stay deleted
  - `synthetic_*` generated materials do not require persisted source rows
  - audit traces exist for proposal totals before PDF generation
  - tax is not double-counted
  - sound mat and rolled rubber route to the intended labor tiers
